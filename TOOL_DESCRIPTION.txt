PLACE VISUALISER — FUNCTIONAL DESCRIPTION
==========================================

Overview
--------
A browser-based geospatial visualisation tool for CSV and JSON/JSONL datasets.
Built with Python/Flask on the backend and plain HTML/JavaScript on the frontend.
Runs locally on http://localhost:5004.

Upload any file with latitude/longitude columns and instantly visualise records
on an interactive OpenStreetMap background. Uses confidence scores to
progressively reveal lower-quality records as the user zooms in.

INTERFACE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Left sidebar — Data Source
  Upload a CSV or JSON/JSONL file (up to 500 MB, max 200 000 rows loaded).
  After upload, column names are listed and lat/lng/confidence columns are
  auto-detected from common naming patterns.

Left sidebar — Column Mapping
  · Latitude / Longitude  — select the coordinate columns
  · Confidence            — optional 0–100 score (or 0–1, auto-normalised)
  · Category field        — used to pick the icon for each record
                            (auto-detects: featurename → category → type →
                             any column containing "categor" or "feature")

Left sidebar — Feature Labels
  Select any column to use as a map label. Labels are drawn in black text
  next to each icon. A spatial grid deduplication prevents label crowding —
  one label per 80×80 px cell at any zoom level.
  Auto-detects: poi_name → name → any column containing "name".

Left sidebar — Feature Filter
  After clicking Visualise, a scrollable checkbox list appears showing all
  unique category values (sorted by count descending). Use All / None buttons
  to quickly toggle groups. Only checked categories are rendered on the map.

Left sidebar — Tooltip Fields
  Select any columns to include in the hover tooltip shown on mouse-over.

Left sidebar — Visualise button
  Fetches data from the server and renders it on the map.

Left sidebar — Statistics
  · Total records    — number of records loaded (after NaN-drop)
  · Visible now      — records currently rendered at the current zoom
  · Min confidence   — the active confidence threshold (≥ value shown)
  · Map zoom         — current Leaflet zoom level (decimal)

Map area
  · OpenStreetMap tiles via Leaflet.js
  · Transparent HTML5 Canvas overlay for icons and labels (pointer-events:none)
  · Standard Leaflet pan/zoom controls

Info bar (top-right overlay)
  Always-visible readout of current threshold, visible count, and zoom level.
  Updates in real time as the user pans or zooms.

"No records visible" hint
  Centred overlay shown when the confidence threshold exceeds all loaded
  records. Prompts the user to zoom in.

"Fit to data" button (bottom-left)
  Re-fits the map to the data bounding box and resets baseZoom so the
  confidence zoom curve restarts from the fitted zoom level.

Hover tooltip
  Appears near the cursor when hovering within 25 px of a record.
  Shows: icon, record number, lat/lng, confidence (colour-coded), and any
  checked tooltip fields. Throttled to 40 ms to avoid lag on large datasets.

CONFIDENCE-BASED ZOOM FILTERING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Records are shown or hidden based on their confidence score relative to the
current map zoom level:

  threshold = 100 / 2^((currentZoom − baseZoom) / 2)

  · Records with confidence ≥ threshold are drawn.
  · At the initial zoom level (baseZoom), only records with confidence = 100
    are visible.
  · Every 2 Leaflet zoom levels zoomed in, the threshold halves, gradually
    revealing lower-confidence records.
  · baseZoom is set at load time by fitting the map to the data bounds.
    If the dataset's maximum confidence is below 100, baseZoom is adjusted
    so the highest-confidence records are visible on initial load.

Confidence auto-normalisation (server-side):
  · Values in [0, 1]       → multiplied by 100
  · Values > 100           → min-max scaled to [0, 100]
  · Values already in (1, 100] → used as-is

Left sidebar — Area Viewer
  A second independent section below the point-data controls.
  Upload a CSV file that contains a WKT (Well-Known Text) geometry column.

  · Upload CSV           — up to 50 000 rows; column names returned immediately
  · WKT geometry column  — select the column containing WKT strings
                            (auto-detected: geometry / wkt / geom / shape /
                             polygon / multipolygon / the_geom)
  · Filter by field      — select any property column to colour and filter by
                            (auto-detected: admin_level → any column containing
                             "admin" → first column)
  · Show Areas button    — sends the WKT column to the server; shapely parses
                           each WKT string into a GeoJSON geometry; the result
                           is rendered as a Leaflet GeoJSON layer of coloured
                           semi-transparent polygons. The map auto-fits to the
                           layer bounds on first load.
  · Filter checkboxes    — one row per unique value of the chosen filter field,
                           each with a colour swatch and feature count.
                           Toggling a checkbox hides/shows those polygons
                           in-place (no server round-trip).
  · All / None buttons   — bulk-toggle the filter checkboxes.
  · ↺ Refresh button     — re-fetches from server (use after changing WKT column).
  · Clear Areas button   — removes the layer from the map.
  · Hover tooltip        — shows all properties of the hovered polygon.

TECHNICAL NOTES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  · Language / framework : Python 3, Flask, pandas, shapely
  · Frontend             : Single HTML file (templates/index.html),
                           Leaflet.js 1.9.4 via CDN for map rendering,
                           D3.js v7 (d3.max for baseZoom calculation),
                           HTML5 Canvas for point/label rendering,
                           Leaflet GeoJSON layer for area rendering
  · Port                 : 5004
  · Startup              : python3 app.py
  · Max rows loaded      : 200 000 (point upload); 150 000 (rendered);
                           50 000 (area upload); 10 000 (area rendered)
  · File formats         : CSV, JSON, JSONL / NDJSON (points);
                           CSV with WKT geometry column (areas)
  · Canvas rendering     : Float64Array typed arrays + viewport culling
                           for smooth performance at large record counts
  · Smart re-fetch       : Changing label/category field only triggers a
                           server re-fetch if the field is not already present
                           in loaded records; otherwise redraws in-place
